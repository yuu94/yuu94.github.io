React Hooksの基本と実践的な使い方

React Hooksは関数コンポーネントで状態管理やライフサイクルメソッドの機能を使えるようにする仕組みです。この記事では基本的な使い方から実践的な応用例まで解説します。

目次

  1. useState - 状態管理の基本
  2. useEffect - 副作用の処理
  3. カスタムフックの作成
  4. まとめ

useState - 状態管理の基本

useStateは最もよく使用されるHookの一つです。

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>現在の値: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        増加
      </button>
      <button onClick={() => setCount(count - 1)}>
        減少
      </button>
    </div>
  );
}

export default Counter;

画像埋め込みテスト

ポイント

  • 初期値をuseState(0)のように渡す
  • 戻り値は[状態, 状態更新関数]の配列
  • 状態更新関数は非同期で実行される

注意: 状態の更新は非同期で行われるため、すぐに新しい値が反映されるわけではありません。

useEffect - 副作用の処理

useEffectはコンポーネントの副作用（API呼び出し、DOM操作など）を処理するためのHookです。

import React, { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const fetchUser = async () => {
      try {
        setLoading(true);
        const response = await fetch(`/api/users/${userId}`);
        const userData = await response.json();
        setUser(userData);
      } catch (error) {
        console.error('ユーザー情報の取得に失敗しました:', error);
      } finally {
        setLoading(false);
      }
    };

    if (userId) {
      fetchUser();
    }
  }, [userId]); // userIdが変更されたときのみ実行

  if (loading) {
    return <div>読み込み中...</div>;
  }

  return (
    <div>
      <h2>{user?.name}</h2>
      <p>{user?.email}</p>
    </div>
  );
}

useEffectの依存配列パターン

パターン実行タイミング
useEffect(fn)毎レンダリング後
useEffect(fn, [])初回レンダリング後のみ
useEffect(fn, [dep])depが変更された時
